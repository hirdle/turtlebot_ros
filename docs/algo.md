## Переменные и функции

### Словарь colors
Содержит нижние и верхние границы всех цветов, кроме красного, в формате HSV.

### Функция get_mask - получение маски определенного цвета
Аргументы - исходное изображение и цвет, по которому делаем маску

1. Преобразование изображения в формат HSV (cvtColor)
2. Задаем сглаживание маски: массив, заполненный единичками 5x5 с uint8 форматом (kernel)
3. Для НЕ красного цвета
- получаем нижнюю и верхнюю границу цвета
- создаем маску, используя hsv изображение и границы (inRange)
4. Для красного цвета
- создаем две маски по двум диапазонам
np.array([0, 100, 100]), np.array([10, 255, 255])
np.array([160, 100, 100]), np.array([180, 255, 255])
- создаем общую маску с их объединением (bitwise_or)
5. Производим сглаживание маски с MORPH_OPEN и MORPH_CLOSE с kernel (morphologyEx)
6. Возвращаем маску

### Функция find_contour - нахождение максимального контура в маске
Аргументы - маска и минимальная площадь контура

1. С параметрами RETR_EXTERNAL, CHAIN_APPROX_SIMPLE находим контуры на маске (findContours)
2. Фильтруем все контуры, площадь которых (contourArea) меньше минимального
3. Возвращаем максимальный контур если он есть

### Функция find_dominant_color - поиск преимущественного цвета
Аргумент - исходное изображение

1. Идем циклом по цветам
- проверяем маски цветов (countNonZero)
- если площадь больше прошлого максимального, обновляем перменные
2. Возвращаем цвет

### Функция crop_roi - выделение фрагмента для работы
Аргументы - исходное изображение, контур для вырезки, отступы и размер

1. делаем reshape контура с параметрами -1, 2 (pts)
2. находим x1 и y1 с pts.min(axis=0) - pd
3. находим x2 и y2 с pts.max(axis=0) + pd
4. находим высоту и ширину исходного с .shape[:2]
5. находим x1 и y1 как max(0, x1) и max(0, y1)
6. находим x2 и y2 как min(w, x2) и min(0, y2)
7. Берем контур из исходного в int [y1:y2, x1:x2]
8. Делаем resize с аргументами из п.7 если её размер больше нуля

### Функция detect_sign - сопоставление шаблона с изображением
Аргументы - изображение, список путей к шаблонам, пороговое значение определения (~0.4-0.45)

0. если кадра нет, возвращаем (False, 0.0, None)
1. определяем цвет (dominant_color, если color='auto')
2. создаем маску цвета из п.1
3. получаем контур из find_contour, при его отсутствии — False
4. получаем целевой объект через crop_roi с аргументами изображения и контура, если roi пустой — False
5. преобразовываем целевой участок в оттенки серого с размытием по гауссу (9)
6. создаем переменные best_name и best_conf
7. идем циклом по путям шаблонов
- подгружаем изображение через imread, пропускаем пустые
- обрабатываем шаблон действиями из п. 2-5 (маска, контур, roi, перевод в серый + блюр 5x5)
- приводим размер шаблона к размеру roi_gray (resize)
- считаем совпадение matchTemplate с TM_CCOEFF_NORMED, берем максимум через minMaxLoc
- если уверенность выше текущей — обновляем best_conf и best_name (basename пути)
8. возвращаем кортеж (best_conf >= threshold, best_conf, best_name)
